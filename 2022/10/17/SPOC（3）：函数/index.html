函数的参数传递（Parameter passing of functions）

形式参数是不占内存的，当调用函数的时候，才会给形参分配内存空间。
实参可以是常量，变量和表达式。
转化（隐含）：将实参转化为形参。
单向传值的作用，被调用函数只会处理形参，形参得到形参的值的时候，就切割了。
return语句只能返回一个值。
现在需要一个双向传值的事物，方法是传引用&。传引用作参数可以保障实参数据的安全。

Formal arguments do not occupy memory, and when the function is called, memory space is allocated to the form parameters.

Arguments can be constants, variables, and expressions.

Transformation (Implicit): Converts the real parameter into a form parameter.

The function of one-way value transfer is that the called function will only process the form parameters, and when the form parameters get the value of the form parameters, they will be cut.

The return statement can return only one value.

Now you need a thing that passes values in both directions, and the way to do this is by passing a reference &. Passing reference parameters can ensure the security of the parameter data.

引用类型（Reference type）

值传递：单向传递。
引用传递：双向传递。
引用类型、
引用（&）是标识符的别名
定义一个引用的时候，必须同时对它进行初始化，使它指向一个已存在的对象。

Value passing: One-way passing.

Reference passing: Two-way delivery.

Reference type,

A reference (&) is an alias for an identifier

When defining a reference, it must also be initialized so that it points to an existing object.

例如（for instance）：
int i，j;
int &ri=i;//定义int引用ri，并初始化为变量i的引用。
j=10;
ri=j;//相当于把j赋值给i（It is equivalent to assigning j to i）
一旦一个引用被初始化后，就不能改为指向其他对象
引用可以作为形参

Once a reference has been initialized, it cannot be changed to point to another object

References can be used as form parameters

含有可以变化的参数的函数（A function with parameters that can be varied）

可以变化长度的参数表：
如果所有的实参类型相同，可以传递一个initializer_list的标准库类型
如果实参的类型不同，我们可以编写可变参数的模板

Table of parameters that can vary in length:

If all arguments are of the same type, you can pass a initializer_list standard library type

If the types of arguments are different, we can write a variadic template initializer_list
initializer_list是一种标准库类型，用于表示某种特定类型的值的数组，该类型定义在同名的头文件中。

initializer_list is a standard library type used to represent an array of values of a particular type, defined in a header file with the same name.

具体操作（How to do it:）：
initializer_listlst;//默认初始化，T类型元素的空列表（Default initialization, an empty list of T-type elements）
nitializer_listlst{a,b,c};//lst的元素数量和初始值一样多，lst的元素是对应初始值的副本；列表中的元素是const（The number of elements in the LST is as large as the initial value, and the element in the LST is a copy of the corresponding initial value; The element in the list is const）
lst2(lst)//拷贝或者赋值一个initializer_list对象但不拷贝列表中的元素；拷贝后原始列表和副本共享元素（Copy or assign an initializer_list object without copying the elements in the list; The original list and copy share elements after copying.）。
lst2=lst
lst.size(）//列表中的元素数量（The number of elements in the list）
lst.begin（）//返回指向lst首元素的指针（Returns a pointer to the LST header element）
lst.end()//返回指向lst尾元素下一位置的指针（Returns a pointer to the next position of the LST tail element）
在编写代码输出程序所产生的错误信息时，最好统一用一个函数实现该功能，使得对所有错误的处理都能够整齐划一，
然而错误信息的种类不同，
调用错误星系时输出函数时传递的参数也会各不相同
使用initializer_list编写一个错误信息输出函数，使其可以作用于可变数量的形参。

When writing the error message generated by the code output program, it is best to implement the function with a unified function, so that all errors can be handled uniformly.

However, the types of misinformation are different,

When calling the error galaxy, the parameters passed in the output function will also vary

Use initializer_list to write an error message output function that can act on a variable number of parameters.

Reduce the hassle of calling functions.

函数重载(Function overloading)

函数多态性的体现，C++中允许功能相近的函数在相同的作用域内以相同函数名声明，从而形成重载。方便使用，便于记忆。

The embodiment of function polymorphism, C++ allows functions with similar functions to be declared with the same function name in the same scope, thus forming an overload. Easy to use and easy to remember.

注意事项，重载函数的形参必须不同：个数不同或者类型不同。
编译程序将根据实参和形参的类型及个数的最佳匹配来选择调用哪一个函数。

Note that the parameters of the overloaded function must be different: the number of parameters or the type of the function must be different.

The compiler will choose which function to call based on the best match between the type and number of arguments.

编译器不以形参名来区分

Compilers do not distinguish by parameter names

编译器不以返回值来区分
不要将不同功能的函数声明为重载函数，以免出现调用结果的误解，混淆，这样不好。

The compiler does not distinguish by the return value

Don’t declare functions with different functions as overloaded functions, so as to avoid misunderstanding and confusion of the call results, which is not good.
